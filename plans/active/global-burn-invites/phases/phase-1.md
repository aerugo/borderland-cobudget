# Phase 1: Data Model & Settings Mutation

**Status**: Pending
**Started**: —
**Parent Plan**: [development-plan.md](../development-plan.md)

## Objective

Persist Global Burn connection settings (instance URL, event ID, API key,
verified flag) on the `Round` model and expose a mutation to edit them.
The API key must never be returned through GraphQL — only a status enum.
No UI in this phase; verification via GraphQL playground.

## Implementation Steps

### Step 1.1: Prisma schema changes

Modify [ui/server/prisma/schema.prisma](../../../ui/server/prisma/schema.prisma),
`Round` model (lines 210-278). Add near the existing `ocToken` /
`ocVerified` fields for grouping:

```prisma
globalBurnInstanceUrl  String?
globalBurnEventId      String?
globalBurnApiKey       String?
globalBurnVerified     Boolean?
```

All fields are nullable; absence means "not configured". No defaults.

Run the migration:

```bash
cd ui && yarn migrate
```

Name the migration `global_burn_settings` when prompted.

### Step 1.2: GraphQL SDL updates

Modify [ui/server/graphql/schema/index.js](../../../ui/server/graphql/schema/index.js).

Add to the `Round` type (near `ocTokenStatus`, `ocVerified`):

```graphql
globalBurnInstanceUrl: String
globalBurnEventId: String
globalBurnApiKeyStatus: TOKEN_STATUS
globalBurnVerified: Boolean
```

Note: `globalBurnInstanceUrl` and `globalBurnEventId` are non-secret — OK
to return. `globalBurnApiKey` field is NOT added to the GraphQL type —
only the status is.

Add the mutation:

```graphql
editGlobalBurnSettings(
  roundId: ID!
  instanceUrl: String
  eventId: String
  apiKey: String
): Round
```

All arguments optional except `roundId`. Semantics per argument:
- `null` → field unchanged
- `""` (empty string) → clear the field (set to null in DB)
- non-empty string → set the field

### Step 1.3: Type resolver for `globalBurnApiKeyStatus`

Modify [ui/server/graphql/resolvers/types/Round.ts](../../../ui/server/graphql/resolvers/types/Round.ts).

Add next to the existing `ocTokenStatus` resolver (around line 373):

```ts
globalBurnApiKeyStatus: async (round, _, { user }) => {
  // Gate to admins only, same as ocTokenStatus
  // Match the existing pattern — check isCollAdmin / group admin / ss
  return round.globalBurnApiKey ? "PROVIDED" : "EMPTY";
},
```

Non-secret fields (`globalBurnInstanceUrl`, `globalBurnEventId`,
`globalBurnVerified`) are direct Prisma column reads — no custom resolver
needed.

### Step 1.4: Mutation resolver

Add to [ui/server/graphql/resolvers/mutations/round.ts](../../../ui/server/graphql/resolvers/mutations/round.ts)
alongside the other edit mutations (e.g. near `editOCToken`):

```ts
export const editGlobalBurnSettings = combineResolvers(
  isCollOrGroupAdmin,
  async (_, { roundId, instanceUrl, eventId, apiKey }) => {
    const data: Record<string, string | null | boolean> = {};

    if (instanceUrl !== null && instanceUrl !== undefined) {
      data.globalBurnInstanceUrl = instanceUrl === "" ? null : instanceUrl;
    }
    if (eventId !== null && eventId !== undefined) {
      data.globalBurnEventId = eventId === "" ? null : eventId;
    }
    if (apiKey !== null && apiKey !== undefined) {
      data.globalBurnApiKey = apiKey === "" ? null : apiKey;
    }

    // Any settings change invalidates prior verification
    if (Object.keys(data).length > 0) {
      data.globalBurnVerified = false;
    }

    return prisma.round.update({ where: { id: roundId }, data });
  }
);
```

### Step 1.5: Wire the mutation into the resolver map

Modify [ui/server/graphql/resolvers/index.js](../../../ui/server/graphql/resolvers/index.js)
(or whichever file assembles the `Mutation` resolver map). Add
`editGlobalBurnSettings` to the `Mutation` object next to the other round
mutations.

## Edge Cases to Handle

- Empty string vs `null` on `apiKey`: empty clears, null leaves alone.
  UI must pass `null` when the password field hasn't been touched.
- Trim URL + event ID on write (strip trailing whitespace / slashes on
  `instanceUrl`) — do it in the resolver so every call path is covered.
- Round does not exist → Prisma `update` throws; let it propagate. The
  `isCollOrGroupAdmin` guard runs first and will reject non-admins before
  the update attempt.

## Verification

```bash
cd ui

# Type check
yarn typecheck

# Run dev server and test via GraphQL playground
yarn dev
```

Manual playground tests:

1. As a round admin, run:
   ```graphql
   mutation {
     editGlobalBurnSettings(
       roundId: "<ROUND_ID>"
       instanceUrl: "https://members.theborderland.se"
       eventId: "2026"
       apiKey: "test-key"
     ) {
       id
       globalBurnInstanceUrl
       globalBurnEventId
       globalBurnApiKeyStatus
       globalBurnVerified
     }
   }
   ```
   Expect: `instanceUrl`, `eventId` echoed, `apiKeyStatus = PROVIDED`,
   `verified = false`.

2. Confirm the raw `globalBurnApiKey` is **not** queryable:
   ```graphql
   query { round(id: "...") { globalBurnApiKey } }
   ```
   Expect: GraphQL schema error (field not on type).

3. Clear the key:
   ```graphql
   mutation {
     editGlobalBurnSettings(roundId: "...", apiKey: "") {
       globalBurnApiKeyStatus
     }
   }
   ```
   Expect: `apiKeyStatus = EMPTY`.

4. Update only the URL (apiKey: null / omitted):
   ```graphql
   mutation {
     editGlobalBurnSettings(roundId: "...", instanceUrl: "https://other") {
       globalBurnInstanceUrl
       globalBurnApiKeyStatus
     }
   }
   ```
   Expect: URL updated, `apiKeyStatus` still `PROVIDED` from step 1.

5. Confirm admin gate: as a non-admin user, attempt the mutation. Expect
   a not-authorized error (same message as other admin-gated mutations).
